Ran a type audit on the current design system. We have six different font sizes in use, but they don't form a legible scale — they grew organically over 18 months and it shows. Some headers are barely larger than body copy.

Rebuilt the scale from scratch using a 1.25 ratio (major third). Starting at 12px for captions, this gives a clean set of stops: 12 / 15 / 19 / 24 / 30 / 37. Tested in context at both 13" and 27" screen sizes.

![Type scale specimen](/images/type-study.svg)

The monospaced constraint is doing interesting work here. Because all characters are the same width, the eye reads columns as columns — you can align things without explicit grid lines. Numbers in tables suddenly feel intentional rather than approximate.

One thing I keep coming back to: **optical sizing**. At 12px, IBM Plex Mono reads clearly but feels compact. At 30px it starts to feel loose. The tracking adjustments that work at small sizes actively hurt legibility at display sizes. Will need responsive tracking.

**Changes going into the system:**

- Replace ad-hoc size tokens with the six-stop scale
- Add a `--tracking-tight` / `--tracking-loose` pattern that inverts by size
- Drop the one-off 16px and 18px tokens — they'll be absorbed by the scale
